Help

Golden path (no code)

  1. Create a strategy (Guided or English)
  2. Get data from Data (download / consolidate)
  3. Open Strategies → edit spec YAML → Save + compile
  4. Run a backtest from Backtest
  5. Inspect results in Runs (logs + report links)
  6. Use “Rerun (same settings)” on a run to iterate fast

Research → validation → live (typical)

  1. Backtest quickly (does the idea look promising?)
  2. Run Phase 1/2 validation + final OOS validation from scripts/ (reduce overfitting risk)
  3. If the target execution venue is:
    • Crypto: trade live directly from TradingLab (API adapters)
    • CME futures / FX: export to the platform (NT8 / MT5), then re-backtest with platform semantics before going live
Tip: treat cross-platform agreement as a strong sanity check, not a guarantee.

Run the GUI (local)

./.venv/bin/python -m pip install -r requirements-gui.txt
./.venv/bin/python scripts/run_gui_launcher.py --host 127.0.0.1 --port 8000
Artifacts (inputs/command/stdout/stderr) are stored in data/logs/gui_runs/.

Common pitfalls

  • Split policy conflicts: if you provide custom --data/start/end, use split policy mode none or enable override.
  • Empty or weird backtests: check the run’s Stdout/Stderr in Runs and confirm your dataset date range matches the UI inputs.
  • Spec edits not reflected: after editing a spec, click “Save + compile”, then rerun the backtest.
  • Reporting a bug: open the run detail page and click “Export bug report (zip)”.
